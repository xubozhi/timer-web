语音标记计时器

timer-web —— 一款轻量无依赖的纯前端PWA语音标记计时器，支持语音/手动双模式标记时间，离线可用、可安装到设备，所有数据本地存储，无后端、零部署成本。

提交者：xubozhi
仓库地址：https://github.com/xubozhi/timer-web

核心功能

✅ 「语音+手动」双标记：原生Web Speech语音控制，不支持时可手动点击标记，兼顾便捷性与兼容性
✅ 全功能语音指令：开始/暂停/标记/重置/导出，纯前端离线识别，无服务器请求
✅ 标记管理：支持编辑标记说明、删除单条记录，标记时间精准同步计时进度
✅ 本地导出：标记记录一键导出为CSV，方便后续整理分析，文件本地生成不上传
✅ PWA离线+安装：Service Worker实现离线缓存，无网络正常使用；支持安装到手机/电脑桌面，脱离浏览器运行
✅ 自适应多端：适配移动端/桌面端，支持系统浅色/深色模式自动切换，交互贴合设备操作习惯
✅ 实时状态反馈：清晰展示计时、识别、权限、离线等状态，操作结果即时提示

技术栈

• 原生HTML/CSS/JavaScript（无任何框架，轻量简洁，无依赖）

• PWA（manifest.json + Service Worker）：实现应用安装、离线缓存

• 原生Web Speech API：前端离线语音识别，无第三方接口依赖

• 原生Blob/URL API：本地生成CSV文件，实现无后端导出

• 响应式布局：clamp自适应字体、弹性布局、媒体查询，适配全尺寸设备

快速使用

方式1：本地运行

1. 克隆本仓库到本地
git clone https://github.com/xubozhi/timer-web.git
cd timer-web
2. 启动本地静态服务器（语音识别/Service Worker需http/https环境，不可直接打开HTML）

◦ 推荐：VS Code插件「Live Server」一键启动

◦ 或Node.js静态服务器：
npm install -g serve
serve .
3. 打开浏览器访问启动地址（如http://localhost:3000）即可使用

方式2：部署上线

将项目所有文件直接上传至任意静态托管平台即可，无需配置后端，推荐：

• GitHub Pages（本仓库可直接开启）

• Vercel / Netlify（一键部署，无需手动操作）

• Gitee Pages / 阿里云/腾讯云静态托管

方式3：直接使用（若开启GitHub Pages）

访问仓库Pages地址：https://xubozhi.github.io/timer-web

项目结构
timer-web/
├── manifest.json  # PWA配置：应用名称、图标、启动方式、主题色等
├── sw.js          # Service Worker：离线缓存、资源请求拦截、缓存更新
├── index.html     # 主文件：整合HTML结构、CSS样式、JS核心逻辑，无外部依赖
└── icon-*.png     # 多尺寸应用图标：适配不同设备/安装场景的图标展示
注：主文件中已内嵌图标Base64，外部图标文件为PWA安装时的兜底适配，可按需保留。
注意事项

1. 语音识别兼容性：仅支持实现Web Speech API的浏览器（Chrome/Edge/Opera等），Safari/iOS Safari暂不支持；不支持时语音功能自动禁用，可正常使用手动标记。

2. 麦克风权限：首次使用语音控制需允许浏览器获取麦克风权限，拒绝后可在浏览器「设置-网站权限」中重新开启。

3. 运行环境：必须在http/https环境下运行，本地开发需用静态服务器，直接打开HTML文件会导致语音识别、Service Worker失效。

4. 数据存储：所有计时、标记数据均保存在浏览器内存中，刷新页面后重置，导出的CSV文件为本地唯一持久化方式。

5. 离线功能：首次在线访问后，浏览器会缓存所有资源，后续无网络环境下可正常使用全部功能。

适配设备/浏览器

支持的设备

• 移动端：安卓/鸿蒙手机/平板（支持PWA的浏览器）

• 桌面端：Windows/Mac/Linux电脑

• 可安装为PWA应用：安卓手机添加到桌面、Windows/Mac通过Chrome/Edge安装为桌面应用

推荐浏览器

Chrome ≥ 80、Edge ≥ 80、Opera ≥ 67（均为原生支持Web Speech API和PWA全特性）
